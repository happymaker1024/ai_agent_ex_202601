## 2. 백엔드 테스트

### 2.1. 서버 시작

```bash
cd backend
uv run python main.py
```

### 2.2. API 엔드포인트 테스트

#### 루트 엔드포인트 확인

브라우저에서 접속: http://localhost:8000

예상 응답:
```json
{
  "message": "투자 레포팅 Agent API",
  "version": "2.0.0",
  "mode": "Demo Mode",  // 또는 "CrewAI-powered"
  "endpoints": {
    "home": "/",
    "invest_report": "/invest_report"
  }
}
```

#### 투자 레포트 요청 (curl)

```bash
curl -X POST http://localhost:8000/invest_report \
  -H "Content-Type: application/json" \
  -d '{"company": "삼성전자"}'
```

#### 투자 레포트 요청 (PowerShell)

```powershell
Invoke-RestMethod -Method Post -Uri "http://localhost:8000/invest_report" `
  -ContentType "application/json" `
  -Body '{"company": "삼성전자"}'
```

### 2.3. API 문서 확인

브라우저에서 접속: http://localhost:8000/docs

- Swagger UI에서 직접 API를 테스트할 수 있습니다.

## 3. 프론트엔드 테스트

### 3.1. 개발 서버 시작

```bash
cd frontend
npm start
```

브라우저에서 자동으로 http://localhost:3000 이 열립니다.

### 3.2. UI 테스트 시나리오

#### 시나리오 1: 더미 데이터 모드 (OpenAI API 키 없음)

1. 드롭다운에서 "삼성전자" 선택
2. "레포팅 요청" 버튼 클릭
3. 로딩 화면 확인:
   - 스피너 애니메이션
   - 4개 에이전트 단계 표시
   - "이 작업은 30초~1분 정도 소요될 수 있습니다" 메시지
4. 레포트 확인:
   - 회사명: 삼성전자
   - 산업: 반도체/전자
   - 주가 정보 (현재가, 전일대비, 거래량, 시가총액)
   - 재무 정보 (매출액, 영업이익, 순이익, 영업이익률)
   - 투자 포인트 3개
   - 리스크 요인 3개
   - 목표주가 및 투자의견
5. "← 돌아가기" 버튼으로 초기 화면 복귀 확인

#### 시나리오 2: CrewAI 모드 (OpenAI API 키 있음)

**사전 준비:**
```bash
cd backend
cp .env.example .env
# .env 파일을 열어서 OPENAI_API_KEY 입력
```

1. 백엔드 서버 재시작
2. 프론트엔드에서 "SK하이닉스" 선택
3. "레포팅 요청" 버튼 클릭
4. 로딩 화면 표시 (약 30초~1분)
5. CrewAI 에이전트가 생성한 실시간 분석 결과 확인:
   - FinanceDataReader로부터 가져온 실제 주가 데이터
   - LLM이 생성한 투자 포인트
   - 실시간 시장 분석 결과
   - 목표주가 및 투자의견

#### 시나리오 3: 다양한 종목 테스트

다음 종목들을 순차적으로 테스트:
- 삼성전자
- SK하이닉스
- 현대차
- NAVER
- 카카오
- LG에너지솔루션

각 종목마다 레포트가 제대로 표시되는지 확인합니다.

#### 시나리오 4: 에러 처리 테스트

1. 백엔드 서버를 종료
2. 프론트엔드에서 레포팅 요청
3. 에러 메시지 표시 확인:
   "레포트를 가져오는데 실패했습니다. 백엔드 서버를 확인해주세요."

## 5. 성능 테스트

### 5.1. 응답 시간

- **더미 데이터 모드**: < 1초
- **CrewAI 모드**: 30초 ~ 60초
  - 재무 분석가: 약 10초
  - 시장 분석가: 약 10초
  - 리스크 분석가: 약 10초
  - 투자 자문가: 약 10초

### 5.2. 동시 요청 테스트

여러 브라우저 탭에서 동시에 다른 종목을 요청하여 테스트할 수 있습니다.

## 6. 문제 해결

### 백엔드 시작 안됨

```bash
# 가상환경 재생성
cd backend
rmdir /s .venv  # Windows
rm -rf .venv    # macOS/Linux

uv venv
uv pip install -r requirements.txt
```

### 프론트엔드 빌드 오류

```bash
cd frontend
rm -rf node_modules
npm install
```

### CORS 오류

- 백엔드가 실행 중인지 확인: http://localhost:8000
- main.py의 CORS 설정 확인:
  ```python
  allow_origins=["http://localhost:3000"]
  ```

### CrewAI 에이전트가 작동하지 않음

1. `.env` 파일에 OPENAI_API_KEY가 올바르게 설정되었는지 확인
2. OpenAI API 키에 충분한 크레딧이 있는지 확인
3. 백엔드 콘솔에서 에러 메시지 확인
